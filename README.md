# Simple CTF

A dockerized Capture The Flag web application with an Obsidian Protocol–themed React UI. Display a flag in the browser, customize all visible copy via environment variables, and retrieve the flag from inside the container CLI.

Dependencies are installed **only inside the Docker image** at build time. No `npm install` on the host is required.

## Quick start

```bash
docker build -t simple-ctf .
docker run -p 3000:3000 simple-ctf
```

Open [http://localhost:3000](http://localhost:3000).

## Architecture

```text
docker run (-e FLAG, -e CTF_*)
        │
        ▼
docker-entrypoint.sh  ──► sets/generates FLAG, writes /etc/ctf-flag.env
        │
        ▼
Express (port 3000)
  ├── GET /api/flag    → { "flag": "..." }
  ├── GET /api/config  → UI strings from env (defaults if unset)
  └── static/          → React build (dist/)
```

- **React + Vite** — frontend
- **Express** — API and static file server
- **docker-entrypoint.sh** — flag generation and shell persistence

## Flag configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `FLAG` | Auto-generated at startup | Flag value shown in the UI and exposed in the container |
| `PORT` | `3000` | HTTP listen port |

### Provided flag

```bash
docker run -p 3000:3000 -e FLAG=myflag simple-ctf
```

### Autogenerated flag

Omit `FLAG`. The entrypoint generates `CTF{<10 lowercase hex chars>}` once per container lifetime:

```bash
docker run -p 3000:3000 simple-ctf
```

Example: `CTF{a8f4b29c1d}`

### Retrieve flag from container CLI

The app process and entrypoint always have `FLAG`. For `docker exec` shells, the flag is also written to `/etc/ctf-flag.env` and `/etc/profile.d/ctf-flag.sh`.

```bash
docker ps

# Login shell (sources profile.d)
docker exec <container_id> sh -lc 'echo $FLAG'

# Non-login shell
docker exec <container_id> sh -c '. /etc/ctf-flag.env && printenv FLAG'

# When you passed -e FLAG=... at docker run
docker exec <container_id> printenv FLAG
```

All UI copy uses the `CTF_` prefix. If unset, the default in the **Default** column is used.

|----------|---------|
| `CTF_APP_TITLE` | `Simple CTF` |

### Customized run example

```bash
docker run -p 3000:3000 \
  -e FLAG=CTF{custom} \
  -e CTF_APP_TITLE="ACME Vault" \
  -e CTF_PAGE_TITLE="ACME Vault // Secure Channel" \
  -e CTF_HEADING_SUCCESS="Access granted // payload ready" \
  -e CTF_COPY_BUTTON="COPY PAYLOAD" \
  simple-ctf
```

## Design

UI follows the **Obsidian Protocol** design system: dark glass surfaces, electric cyan accents, acid green for data highlights, Sora + JetBrains Mono typography, and sharp (0px) corners. Copy is loaded at runtime from `/api/config` so operators can rebrand deployments without rebuilding the image.
